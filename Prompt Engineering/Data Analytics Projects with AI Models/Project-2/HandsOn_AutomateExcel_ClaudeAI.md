# ✅ SECTION 1 — Workshop Overview

## ☑️ 4.3 Understanding the KPI Formulas

The KPI cards pull live values using `GETPIVOTDATA` formulas linked to the Pivot sheet:

| KPI Card | Formula Used | What it Calculates |
|---|---|---|
| Total Sales | `=GETPIVOTDATA("Total Sales",Pivot!$D$2)` | Sum of all sales revenue |
| Total Profit | `=GETPIVOTDATA("Total Profit",Pivot!$D$2)` | Sum of all profit |
| Total Orders | `=COUNTA(Pivot!Y:Y)` | Count of order records |
| Total Qty Sold | `=GETPIVOTDATA("Quantity",Pivot!$AB$2)` | Sum of all units sold |
| Avg Discount | `=GETPIVOTDATA("Avg Discount",Pivot!$U$2)` | Average discount rate applied |
| Profit Margin | `=D5/B5` | Profit as % of Total Sales |

# ✅ SECTION 5 — Bonus: One-Click Refresh Dashboard

## ☑️ 5.1 Why Refresh Matters

When new data rows are added to the Data sheet, the Pivot Tables do not update automatically. The Refresh macro solves this by looping through every worksheet and updating every Pivot Table in one click — then rebuilding the dashboard.

---

## ☑️ 5.2 The Refresh VBA Code

This bonus code is available and can be added as a separate module (if it is not already generated) and can be executed as before:

```vba
Option Explicit

Sub RefreshAllPivots()
    Dim ws As Worksheet
    Dim pt As PivotTable

    For Each ws In ThisWorkbook.Worksheets
        For Each pt In ws.PivotTables
            pt.RefreshTable
        Next pt
    Next ws

    MsgBox "Data & Dashboard refreshed successfully.", _
           vbInformation, "Refresh Complete"
End Sub
```

---

## ☑️ 5.3 How to Wire the Refresh Button

| Step | Action | Details |
|---|---|---|
| 1 | Open VBA Editor | Developer → Visual Basic |
| 2 | Insert Module | Insert → Module → Rename to: `RefreshAll` |
| 3 | Paste Code | Paste the `RefreshAllPivots` code above |
| 4 | Insert Shape | Go back to Excel → Insert a button on any sheet |
| 5 | Label it | Right-click → Edit Text → type: `Refresh Dashboard` |
| 6 | Assign Macro | Right-click → Assign Macro → `RefreshAllPivots` → OK |
| 7 | Test it | Add a dummy row to Data, then click Refresh — check if Pivot & charts update |

# ✅ SECTION 6 — Key Concepts & Learner Notes

## ☑️ 6.1 Core Concepts Covered

| Concept | Explanation |
|---|---|
| **VBA (Visual Basic for Applications)** | A programming language built into Excel that lets you automate repetitive tasks and build complex workflows without writing formulas manually. |
| **Macro** | A saved set of VBA instructions. When you 'assign a macro' to a button, clicking the button runs those instructions automatically. |
| **Module** | A container inside the VBA editor where code is stored. Each phase of this project uses a separate module for organization. |
| **Pivot Table** | An Excel feature that summarises large datasets into compact, interactive summary tables — grouping by segments, dates, categories, etc. |
| **GETPIVOTDATA** | An Excel formula that retrieves specific values from a Pivot Table by name. More reliable than direct cell references when pivot layouts change. |
| **Prompt Engineering** | The skill of writing clear, specific instructions for an AI model (Claude) to get high-quality, usable output — here: working VBA code. |
| **Dashboard** | A visual summary sheet that combines KPI cards and charts, giving decision-makers a single-screen view of business performance. |

---

## ☑️ 6.2 Common Errors & Fixes

| Error / Issue | Likely Cause | Fix |
|---|---|---|
| Macro not found when assigning | Code not saved in the right module | Re-paste the code; ensure the module is in 'This Workbook', not 'Sheet1' |
| 'Pivot' sheet not found error | Pivot sheet was not created first | Run Phase 2 (Create Pivot Tables) before Phase 3 (Dashboard) |
| KPI card shows `#REF!` or 0 | GETPIVOTDATA formula points to wrong pivot cell | Re-run the pivot macro to recreate pivot at the expected cell reference |
| Charts overlap each other | Sheet row heights differ from defaults | Reset row heights: select all rows → right-click → Row Height → set to default |
| Button click does nothing | Macro not assigned correctly | Right-click button → Assign Macro → re-select the correct Sub name |
| Date column unchanged | Column not detected as date type | Ensure the Order Date column contains actual Excel date values, not text |

## ☑️ 6.3 Prompting Tips for Better AI Output

- Be specific about sheet names — Claude uses them literally in the VBA code
- Mention the expected output format (e.g., side-by-side, purple theme, no overlapping)
- If the code has an error, paste the error message back into Claude and ask it to fix it
- Ask Claude to add comments to explain each section if you want to learn the code
